# Product Copy Guide

KanaMe copy should be beginner-friendly, clear, and non-technical.

## Tone rules

- Use simple phrasing and short sentences.
- Prefer "best match" and "suggested kana" over absolute claims.
- Avoid language that implies linguistic certainty.

## Required disclaimers

- Kana output is an approximation of pronunciation.
- Pronunciation and preferred spelling can vary by person, region, and context.
- Users should be able to edit or choose preferred kana when needed.

## Presenting source and warning fields

- If source metadata exists, label it as informational ("Source used"), not authoritative proof.
- If warnings exist, show calm actionable text ("We could not verify externally; using local transliteration").
- Keep warnings visible but non-alarmist.

## Suggested microcopy

- Single conversion:
  - Input label: "Enter a name"
  - Action: "Convert to Kana"
  - Result helper: "Suggested Katakana and Hiragana"
- Bulk conversion:
  - Input label: "Paste one name per line"
  - Hint: "Bulk mode has a line limit to keep the tool fast and reliable."
  - Action: "Convert Names"
- Copy buttons:
  - "Copy Katakana"
  - "Copy Hiragana"
  - "Copy All"
  - Success toast: "Copied"
- Social handle hint:
  - "For social handles, keep it short and readable."

## Future-facing note

- Preferred kana overrides should be supported as a roadmap feature (documented intent; not implemented by default).
